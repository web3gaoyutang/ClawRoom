# ClawRoom 产品文档

## 1. 产品概述

### 1.1 产品名称
ClawRoom

### 1.2 产品定位
线下活动现场的像素风动态签到大屏系统。用户每完成一次签到，大屏中新增一个像素小人，小人在主题房间内自由移动，持续营造"越来越热闹"的氛围。

### 1.3 核心价值
- 将签到行为转化为"角色进入活动空间"的可视化体验
- 提升现场氛围与拍照传播意愿
- 强化活动品牌识别度

### 1.4 适用场景
- 开源社区活动 / 技术沙龙 / 黑客松
- 展会 / 路演 / 品牌快闪活动
- 公司内部活动

---

## 2. 用户角色

| 角色 | 描述 | 核心需求 |
|------|------|----------|
| 活动参与者 | 签到用户 | 签到后立刻看到反馈，自己的角色出现在大屏 |
| 活动主办方 | 运营/组织者 | 大屏有氛围感，运行稳定，支持品牌定制 |

---

## 3. 核心指标

| 指标 | 目标值 |
|------|--------|
| 签到到大屏反馈时间 | ≤ 2 秒 |
| 大屏连续运行稳定性 | 活动期间无重大故障 |
| 用户签到反馈可感知率 | > 90% |
| 并发角色展示数 | ≥ 200 |
| 主办方满意度 | ≥ 8/10 |

---

## 4. 功能清单

### 4.1 MVP 必做功能

#### F1: 签到数据接入
- 提供 REST API 接收签到事件
- 输入字段：用户ID、昵称、头像(可选)、签到时间、标签(可选)
- 支持去重、重试、断线恢复

#### F2: 像素角色生成
- 基于签到信息自动生成像素角色
- 角色属性：基础身体模板 + 随机衣服颜色/发型/配件
- 同一用户保持角色一致性

#### F3: 入场动效
- 角色从入口处进入房间
- 头顶显示昵称 2~3 秒
- 新角色短暂高亮

#### F4: 场景内移动
- 角色随机巡游、停留、转身
- 避免碰撞和堆叠
- 动作节奏舒缓自然

#### F5: 主题场景
- OpenClaw 品牌风格像素房间
- 包含 Logo、终端、服务器机架、海报墙等装饰元素
- 环境动效（灯光闪烁、屏幕滚动等）

#### F6: 数据展示
- 签到总人数
- 活动名称
- 当前时间
- 最新签到昵称（底部滚动）

#### F7: 大屏适配
- 支持 1080P / 4K
- 浏览器全屏投放
- 横屏主流分辨率适配

### 4.2 后续迭代功能

#### V1.1
- 嘉宾/志愿者特殊皮肤
- 签到里程碑特效
- 房间小彩蛋

#### V1.2
- 多主题房间切换
- 活动品牌定制皮肤
- 角色点击查看昵称

#### V2.0
- 丰富互动行为
- 场景解锁机制
- 活动结束后生成纪念海报/回顾视频

## 6. 交互流程

```
用户扫码签到 → 后端生成签到事件 → WebSocket 推送到大屏
→ 生成像素角色 → 播放入场动画 → 显示昵称/欢迎语
→ 角色进入场景随机活动 → 签到人数更新
```

---

## 7. 人数增长策略

| 人数范围 | 策略 |
|----------|------|
| 0~50 | 每人独立角色，完整行为展示 |
| 50~150 | 每人独立角色，限制活动范围，分区移动 |
| 150+ | 分层展示：核心区活跃角色、次级区简化角色、远景区缩小像素人 |

---

## 8. 空闲态策略

当短时间无新签到时：
- 角色正常游走
- 环境微动效（灯光、屏幕、装置动画）
- 周期性展示欢迎语
- 偶尔触发角色互动动作

---

## 9. 风险与应对

| 风险 | 应对措施 |
|------|----------|
| 人数过多画面混乱 | 分区展示、远近景分层、限制密度 |
| 签到后用户感知不到 | 强化入场动画、昵称弹出、高亮提示 |
| 品牌感弱 | 融入 OpenClaw 主题元素与标识 |
| 长时间运行卡顿 | 控制动画复杂度、资源缓存、优化角色逻辑 |
| 签到数据异常 | 服务端去重、前端幂等处理、异常日志上报 |
