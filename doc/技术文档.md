# ClawRoom 技术文档

## 1. 技术架构概览

```
┌─────────────┐     HTTP/WS      ┌─────────────────┐
│   签到终端    │ ──────────────→ │   Backend (API)  │
│  (扫码/表单)  │                  │   Python/FastAPI │
└─────────────┘                  └────────┬────────┘
                                          │ WebSocket
                                          ▼
                                 ┌─────────────────┐
                                 │  Frontend (大屏)  │
```

## 3. 后端设计

### 3.1 数据模型

#### CheckIn（签到记录）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| user_id | String | 用户唯一标识 |
| nickname | String | 昵称 |
| avatar_url | String? | 头像链接（可选）|
| tag | String? | 标签（嘉宾/志愿者等）|
| checked_in_at | DateTime | 签到时间 |
| created_at | DateTime | 记录创建时间 |

#### Character（像素角色）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| user_id | String | 关联用户ID（唯一）|
| body_type | Integer | 身体模板编号 |
| color_primary | String | 主颜色（衣服）|
| color_secondary | String | 副颜色（配件）|
| hair_style | Integer | 发型编号 |
| accessory | Integer? | 配件编号 |
| created_at | DateTime | 创建时间 |

### 3.2 API 设计

#### REST API

```
POST /api/v1/checkin
  Body: { user_id, nickname, avatar_url?, tag? }
  Response: { id, character }
  说明: 接收签到事件，生成角色，触发 WebSocket 广播

GET /api/v1/checkins
  Query: ?limit=50&offset=0
  Response: { total, items: [...] }
  说明: 获取签到列表

GET /api/v1/checkins/count
  Response: { count }
  说明: 获取签到总人数

GET /api/v1/health
  Response: { status: "ok" }
  说明: 健康检查
```

#### WebSocket

```
WS /ws/screen
  说明: 大屏客户端连接

  服务端推送事件:
  - new_checkin: 新签到事件，包含角色信息
    { type: "new_checkin", data: { checkin, character } }

  - sync: 连接时发送当前所有已签到数据
    { type: "sync", data: { checkins: [...], total: N } }

  - heartbeat: 心跳保活
    { type: "heartbeat" }
```

### 3.3 签到处理流程

```
接收 POST /api/v1/checkin
  → 校验参数
  → 检查 user_id 是否已签到（去重）
  → 若已签到：返回已有角色信息
  → 若未签到：
    → 创建签到记录
    → 生成像素角色（确定性随机：基于 user_id hash）
    → 保存角色信息
    → 通过 WebSocket 广播 new_checkin 事件
    → 返回签到结果
```

### 3.4 角色生成算法

基于 `user_id` 的哈希值进行确定性随机，保证同一用户多次签到角色外观一致：

```python
import hashlib

def generate_character_attrs(user_id: str) -> dict:
    hash_bytes = hashlib.sha256(user_id.encode()).digest()
    return {
        "body_type": hash_bytes[0] % BODY_TYPE_COUNT,
        "color_primary": COLORS[hash_bytes[1] % len(COLORS)],
        "color_secondary": COLORS[hash_bytes[2] % len(COLORS)],
        "hair_style": hash_bytes[3] % HAIR_STYLE_COUNT,
        "accessory": hash_bytes[4] % ACCESSORY_COUNT if hash_bytes[5] > 128 else None,
    }
```

## 4. 前端设计

### 4.1 渲染架构

采用 React + PixiJS 混合架构：
- **React** 管理 UI 层（信息面板、标题、底部栏）
- **PixiJS** 管理游戏层（像素房间、角色、动画）

两层通过 React 状态桥接，PixiJS Canvas 作为 React 组件嵌入。

### 4.2 角色移动引擎

```typescript
interface Character {
  id: string;
  nickname: string;
  sprite: PIXI.AnimatedSprite;
  position: { x: number; y: number };
  target: { x: number; y: number } | null;
  state: "idle" | "walking" | "entering";
}

// 移动逻辑
class MovementEngine {
  // 每帧更新
  update(delta: number) {
    for (const char of this.characters) {
      if (char.state === "idle" && Math.random() < 0.005) {
        char.target = this.getRandomWalkablePosition();
        char.state = "walking";
      }
      if (char.state === "walking" && char.target) {
        this.moveToward(char, char.target, delta);
        if (this.reachedTarget(char)) {
          char.state = "idle";
          char.target = null;
        }
      }
    }
  }
}
```

### 4.3 场景地图

使用 Tiled 风格的瓦片地图：
- 地面层：地板瓦片
- 物件层：桌子、服务器机架、海报等（可碰撞）
- 装饰层：灯光、Logo 等（无碰撞）
- 角色层：像素小人

碰撞检测基于简单的网格碰撞图（collision map），标记不可行走区域。

### 4.4 入场动画序列

```
1. 角色精灵在入口处创建（透明度 0）
2. 淡入 + 从门口行走进入（0.5s）
3. 头顶浮现昵称气泡（持续 2.5s）
4. 角色短暂高亮闪烁（0.3s × 3次）
5. 切换为正常状态，开始随机行走
```

### 4.5 WebSocket 连接管理

```typescript
// useWebSocket hook
- 自动连接与重连（指数退避）
- 连接成功后接收 sync 事件，初始化所有已签到角色
- 监听 new_checkin 事件，触发角色入场
- 心跳检测，超时自动重连
```

### 4.6 性能优化策略

| 策略 | 说明 |
|------|------|
| 精灵图集 | 所有角色帧合并为单张 spritesheet，减少 draw call |
| 对象池 | 预创建角色对象，复用而非频繁创建销毁 |
| 视口裁剪 | 仅渲染可视区域内的角色 |
| 帧率控制 | 空闲时降低帧率至 30fps，有新事件时恢复 60fps |
| 分层渲染 | 静态背景单独渲染，不随角色层每帧刷新 |

## 5. 通信协议

### 5.1 WebSocket 消息格式

所有消息均为 JSON 格式：

```json
// 服务端 → 客户端：新签到
{
  "type": "new_checkin",
  "data": {
    "checkin": {
      "id": "uuid",
      "user_id": "user_123",
      "nickname": "张三",
      "tag": "speaker",
      "checked_in_at": "2026-03-18T10:30:00Z"
    },
    "character": {
      "id": "uuid",
      "body_type": 2,
      "color_primary": "#FF6B6B",
      "color_secondary": "#4ECDC4",
      "hair_style": 3,
      "accessory": 1
    }
  }
}

// 服务端 → 客户端：全量同步
{
  "type": "sync",
  "data": {
    "total": 42,
    "checkins": [...]
  }
}

// 服务端 → 客户端：心跳
{
  "type": "heartbeat",
  "timestamp": 1710748200
}
```

## 6. 部署方案

### 6.1 开发环境

```bash
# 后端
cd backend
poetry install
poetry run uvicorn app.main:app --reload --port 8000

# 前端
cd frontend
pnpm install
pnpm dev    # 默认 http://localhost:5173
```

### 6.2 生产部署

```
推荐方案：Docker Compose

┌──────────────────────────────┐
│  Nginx (反向代理 + 静态文件)   │
│  :80 / :443                  │
├──────────┬───────────────────┤
│ Frontend │  Backend          │
│ (静态文件) │  (Uvicorn × N)   │
│          │  :8000            │
└──────────┴───────────────────┘
```

### 6.3 环境变量

| 变量名 | 说明 | 默认值 |
|--------|------|--------|
| DATABASE_URL | 数据库连接字符串 | sqlite+aiosqlite:///./clawroom.db |
